java Object类方法解析

发表于2023-08-04|更新于2023-08-04
|字数总计:1.1k|阅读时长:3分钟|阅读量:

Object类在java中是一个超类，所有类直接或间接的继承的Object类

Object类中的方法

getClass()方法

可以通过这个方法获得类的Class等信息

hashCode() 方法

hashCode() 的作用是获取哈希码（int 整数），也称为散列码。这个哈希码的作用是确定该对象在哈希表中的索引位置。

hashCode() 返回的值并不是一定是地址或者地址的转换。openjdk8中默认使用线程局部状态来实现 Marsaglia’s xor-shift 随机数生成。

equals()方法

在没有重写equals()的情况下，默认的是比较地址，重写后比较属性。

finalize() 方法

对于这个方法已被官方标记为@Deprecated(since="9")，因为可能带来性能问题，当属于GCRoot(可达性分析法)的不可达对象时，将会进行第一次筛选，执行对象的finalize方法(这个过程中可能造成对象复活)，然后将对象放入一个队列进行二次筛选，如果对象还是处于不可达的状态将会被回收。

一些关于Object方法的补充

为什么同时要有hashCode()和equals()两个方法?

我们可以发现hashCode() 和 equals()的作用都是用于比较两个对象是否相等。

我们发现通过hashCode()来两个元素是否相等往往是比 equals()高效的

但是这个国产有可能会发生哈希碰撞(两个不同的对象hashCode相等)。

所以判断两个对象相等往往要进行以下步骤

  1. 如果两个对象的hashCode 值相等，那这两个对象不一定相等（哈希碰撞）。

  2. 如果两个对象的hashCode 值相等并且equals()方法也返回 true，我们才认为这两个对象相等。

  3. 如果两个对象的hashCode 值不相等，我们就可以直接认为这两个对象不相等。

为什么Idea的重写equals()和hashCode()是在一起的?

这个问题在于为什么要同时重写这两个方法，两个对象相等说明两个对象的属性和hashCode都是相等的，如果重写 equals() 时没有重写 hashCode() 方法的话就可能会导致 equals 方法判断是相等的两个对象，hashCode 值却不相等。
